diskutil 製作開機用 USB

發表於 2018-08-18 | 分類於 IT 筆記

[MacOS] 製作開機用 USB碟

Step 1 下載 image檔案 .iso

Step 2 確認 Disk 位置

在 Terminal 底下 使用 diskutil list

diskutil list

來確認磁碟機代號是 /dev/diskN (N表數字)的哪一個

Step 3 Unmount 隨身碟 (讓隨身碟不處在使用狀態中)

diskutil unmountDisk [usb disk location ex: /dev/disk2]

Step 3 使用 dd 指令將 iso 寫入

sudo dd if=/YOURISOFILE.ISO of=[usb disk location ex: /dev/disk2] bs=1m

dd 指令可參考

https://medium.com/@yujiewang/linux-%E6%8C%87%E4%BB%A4%E7%AD%86%E8%A8%98-dd-c873d4e49608

Step 4 退出 USB

diskutil eject /dev/diskN

[Linux] 指令 — dd

data duplicator 低階資料處理工具

基礎作法
dd if=[your_input_file] of=[your_output_file]

—

直接複製整顆硬碟

sudo dd if=/dev/sda of=/dev/sda conv=noerror,sync

使用 watch 顯示 dd 進度

watch -n 5 killall -USR1 dd

空間參數

bs=[BYTES] // 
count=N

ibs=BYTES
obs=BYTES

seek=N
skip=N

conv=CONVS // [ascii,block,lcase,ucase...]

ex

dd if=/dev/sda of=/dev/sdg bs=1G count=22 conv=noerror,sync

—

[docker] 複製 Container 內的檔案到 Host

最近在工作上開始使用到 docker ，
…身為一個 newbie 一定要什麼都來記一下XD

語法很簡單，範例如下

docker cp <containerID>:/file/path/within/container /host/path/target

不過不熟 docker 的人就會跟我一樣還缺兩個東西：

  1. container ID 怎麼來

用 docker ps 就可以列出 container 清單啦

docker ps
  1. 檔案路徑位置怎麼確定

就直接進去看就好

docker exec -t -i <containerID> /bin/bash

參考資料中還有其他 4 種方式，像是建立 snapshot 跟 ssh，
但我覺得實務上還是直接跑進去看比較方便，所以有興趣的朋友再去參考資料看看吧。

使用 VS Code 搭配 Rubocop 養成 coding 整潔習慣

發表於 2018-06-16 | 分類於 IT 筆記

[Rails] 使用 Visual Studio Code 搭配 Rubocop 養成 coding 整潔習慣

基本上，新手寫code總是有些不乾淨的習慣
像是多加空白啦、少加空白啦 縮排沒有對齊等等

Rubocop 就是 在 Ruby 環境中出現的一個工具，可以幫助你檢查有哪些地方不符標準

最原生的作法就是安裝後在檔案存放目錄下執行 rubocop 指令

這樣會出現像是單元測試的檢查結果(如下圖)
到這個部分還是沒什麼用

但是搭配特定文字編輯器 就可以直接在 IDE 中偵測、檢查相關異常
在編輯檔案的時候就直接順手養成良好慣例

綠色底線 像在 Word 中出現的拼字檢查一樣

—

如何在 Visual Studio Code 中安裝、使用 Rubocop

  1. 安裝 Visual Studio Code (省略步驟

  2. 安裝 rubocop

    gem install rubocop

  3. 設定 Visual Studio Code 與 rubocop 連結

在 Extension 列表中搜尋 rubocop

並點選安裝、安裝完成後點選重新載入

點選安裝

點選重新載入

設定 Setting File

點選 [Code] -> [喜好設定] -> [設定]

點選 Setting File — settings.json

加上以下兩行

"ruby.rubocop.executePath": "",

"ruby.rubocop.onSave": true

這樣就大功告成啦！

Rubocop Githup連結
https://github.com/rubocop-hq/rubocop

VS Code 修改 Tab設定

發表於 2018-06-16 | 分類於 IT 筆記

[Rails] Visual Studio Code 修改 Tab設定

寫Ruby的時候依據慣例會習慣使用 2 個空白來設定縮排
但 Visual Studio Code 預設是使用 Tab 來做縮排的，寬度還是 4
這就需要一些調整啦…

點選 狀態列上的 [空白:4] (這邊因為我已經改完才截圖 所以顯示為2)

選擇 [使用空格鍵進行縮排]

然後選擇定位點長度 2 就完成啦

其實這些設定應該是一樣有存成一個config檔的。
之後有研究到再更新摟。

— 2018 06 18 更新 — 結果發現上述的做法只會修正單檔XD

如果套用完整設定的話
先開啟一個 .rb 後，點選狀態列的 ruby 字樣

點選 進行以 ‘Ruby’ 語言為基礎的設定

就會切換到 User Settings的畫面
在使用者設定底下的 “[ruby]”:{} 區域中 加上
“editor.tabSize”: 2 跟“editor.insertSpaces”: true 就好啦

畫面中只加了 tabSize是因為左邊的全域設定都已經是 insertSpaces : true了
所以我就沒加了，如果要加多個設定的話記得在中間加上 , 喔 :D
